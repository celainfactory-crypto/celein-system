## 📋 وصف النظام

نظام سيلين هو حلٌّ متكامل لإدارة مصنع المياه المعدنية والمرطبات، مصمم بواجهة عربية كاملة (RTL) وتصميم عصري بأسلوب **Neumorphism** بألوان هوية سيلين. يعمل كتطبيق **PWA** قابل للتثبيت على الهاتف، مع طبقة بيانات محلية في `localStorage` ونظام مزامنة وتصدير متعدد الصيغ.

### ✨ الميزات الرئيسية

| المجال | التفاصيل |
|--------|-----------|

## 🚀 دليل النشر

### 1. النشر على Netlify (الطريقة الأصلية)

#### المتطلبات
- حساب على [netlify.com](https://www.netlify.com)
- ربط الـ repository الحالي (يحتوي على `netlify.toml`)

#### خطوات النشر

```bash
# 1. تأكد من رفع آخر تعديلات
git add .
git commit -m "نشر إصدار جديد"
git push origin main

# 2. Netlify سيُعيد البناء تلقائياً عند كل push
#    publish = "." كما هو محدد في netlify.toml
```

#### إعدادات Netlify الخاصة بهذا المشروع

ملف `netlify.toml` يُعيد توجيه طلبات `data-seed.js` إلى jsDelivr CDN لتجاوز مشكلة الكاش:

```toml
[build]
  publish = "."

[[redirects]]
  from = "/data-seed.js"
  to = "https://cdn.jsdelivr.net/gh/celainfactory-crypto/celein-system@main/data-seed.js"
  status = 200
  force = true
```

وكذلك ملف `_redirects`:
```
/data-seed.js https://cdn.jsdelivr.net/gh/celainfactory-crypto/celein-system@main/data-seed.js 200
```

#### لماذا CDN؟

الإصدارات `v19.0 → v19.4` انتقلت من استضافة `data-seed.js` على Netlify إلى jsDelivr لأن Netlify كان يحتفظ بنسخة قديمة في الكاش رغم تغيير أسماء الملفات. الحل النهائي:
- `data-seed.js` يُعاد توجيهه إلى jsDelivr (CDN حيّ، تحديث فوري)
- `sw-proxy.js` يعمل كـ Service Worker Proxy
- `celain-direct.html` يحمّل من jsDelivr مباشرة (نسخة بديلة)

### 2. النشر على Cloudflare Pages / بدائل أخرى

```bash
# نسخ المشروع لمجلد منفصل
cp -r /workspace/celein-deploy /tmp/celein-deploy-clean
cd /tmp/celein-deploy-clean

# Cloudflare Pages عبر wrangler
npm install -g wrangler
wrangler pages deploy . --project-name=celain-factory

# أو Surge (بديل سريع)
npm install -g surge
surge . celain-factory.surge.sh
```

### 5. شهادة SSL

- Netlify و Cloudflare Pages: SSL تلقائي (Let's Encrypt)
- النشر اليدوي: استخدم [Caddy](https://caddyserver.com) أو [acme.sh](https://github.com/acmesh-official/acme.sh)

### 6. نظام التحديث التلقائي

النظام يفحص `version.json` كل 60 ثانية. عند نشر إصدار جديد:

1. ارفع `version.json` الجديد (زد رقم الإصدار و`buildTime`)
2. الـ Service Worker (`sw-proxy.js`) يجلب الملف دائماً من الشبكة
3. يظهر شريط إعلاني "تحديث متاح" للمستخدم

```json
{
  "version": "v18.7",
  "buildTime": "2026-07-12T18:36:00Z",
  "message": "تحديث الأسماء: مبروك حمود علي اليوسفي، سالم علي محسن الشتيمي"
}
```

## 💾 النسخ الاحتياطي والاستيراد

### التصدير
- من قائمة الإعدادات → "تصدير JSON"
- يحفظ نسخة كاملة (`window.SEED` + `localStorage` كاملاً)
- اسم الملف الافتراضي: `celein-backup-YYYY-MM-DD.json`

### الاستيراد
- من قائمة الإعدادات → "استيراد من ملف"
- يدمج البيانات أو يستبدلها (حسب الاختيار)
- **تنبيه:** يتم مسح الإصدارات القديمة من `localStorage` تلقائياً قبل الاستيراد (v17.5+)
